## 한줄 요약
바코드 FFW 텔레옵에 녹화 모드를 추가하고, 바코드 타겟/카메라/LeRobot 변환을 데이터 수집 중심으로 확장했다.

## 주요 변경점

- `scripts/teleop_barcode_ffw.py`
  - `--record`, `--dataset_file`, `--num_demos`, `--num_success_steps` 옵션을 추가했다.
  - 성공 조건을 만족하면 HDF5 에피소드를 저장하고 reset하는 녹화 흐름을 넣었다.
  - 녹화 중에는 바코드 타겟 구, 손 마커, 프러스텀 시각화를 숨겨 데이터셋 영상에 인디케이터가 섞이지 않도록 했다.
  - 오른손 프리뷰와 연동되는 네비게이션/카운트 패널 갱신을 추가했다.

- `scripts/isaac_lab_entrypoint.sh`
  - barcode FFW 수집 시 `record_demos.py` 대신 `teleop_barcode_ffw.py --record` 경로로 진입하도록 분기했다.
  - 수집 개수와 성공 유지 스텝을 환경변수로 전달할 수 있게 했다.

- `scripts/convert_isaac_hdf5_to_lerobot_v3.py`
  - HDF5의 카메라 이미지를 자동 탐지하도록 바꿨다.
  - 단일 POV 전용에서 다중 카메라 비디오/메타데이터 생성 방식으로 확장했다.
  - LeRobotDataset import 경로와 출력 디렉터리 처리를 정리했다.

- `scripts/custom_tasks/teleop_barcode_press/barcode_press_env_cfg.py`
  - reference USD에서 바코드 prim을 자동 탐지해 타겟 위치를 계산하도록 바꿨다.
  - 바코드 prim을 찾지 못할 때의 fallback 좌표를 갱신했다.
  - head 카메라 기본 해상도를 데이터셋 영상용 값으로 재조정했다.

- `scripts/custom_tasks/teleop_barcode_press/utils/camera_preview_displays.py`
  - 오른손 카메라 프리뷰 아래에 네비게이션 패널을 추가했다.
  - 목표 방향, 거리, 홀드 타이머, 성공 상태를 시각적으로 보여주도록 했다.

- `docker-compose.yml`
  - head/hand 카메라 해상도 관련 환경변수를 컨테이너에 넘기도록 했다.
